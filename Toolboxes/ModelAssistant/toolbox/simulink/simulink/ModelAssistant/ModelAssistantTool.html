Model Assistant Tool

Using the Model Assistant Tool, you can quickly configure a model for code generation, and identify aspects of your model that impede production deployment or limit code efficiency.  There are four main components of the tool.

 

1.     General code generation goals

2.     Detailed code generation goals

3.     Model advisor

4.     Search and modify

 

Together, these components provide a powerful and centralized interface for configuring settings for Simulink blocks, Stateflow charts, models and subsystems.

 

The Model Assistant Tool can be used at any point during your design cycle, as it is completely separate from the code generation process.

General Code Generation Goals

This component allows you to quickly configure your code generation based on specific goals.  Once you have decided the overall optimization and tradeoffs for your application (e.g., what is more important: RAM or ROM?), the tool will adjust the model settings to best achieve your goals.

 


 


Detailed Code Generation Goals

This component presents a centralized means for configuring your model for code generation.  Options are logically grouped by category, and are applied across products.

 


 


Model Advisor

This component comprehensively analyzes your model to ensure that you best utilize Real-Time Workshop Embedded Coder.  Specific checks can be enabled or disabled at any point.  This is particularly useful early in the design cycle.

 


 


Search and Modify

This component provides powerful search and modify capability.  The goal of the tool is to significantly reduce the burden of block-by-block configuration.  The search aspect helps you quickly find attributes of blocks, lines, input ports, output ports, and annotations.  The modify aspect allows you to quickly perform batch operations on the search results.  Frequently performed tasks are conveniently packaged into a single button click.

 


 


The Search and Modify component has several capabilities for quickly modifying a diagram.

 

The Frequent tasks page allows you to quickly perform common actions.  For example, you can quickly configure the “Saturate on integer overflow” option for blocks in you model, saving you countless button clicks.

 

The Simulink object search page allows you to specify a general Simulink object search and modify action. This search mechanism is useful when you know the specific names of underlying attributes.  For example, to locate all the gain blocks in your model you can specify the attribute BlockType with value Gain.

 

The Stateflow object search page allows you to quickly configure the Stateflow data in your model.  This is particularly useful for converting data from floating point to fixed-point types.  For example, you can quickly extract all double data beginning with d_, and modify them to fixed-point types.

 

The Search and replace Simulink text page allows you to quickly modify text for objects in Simulink.  For example, changing all occurrences of “K1” to “k2”.  The semantics of the search and replace are the same as for the Stateflow search and replace tool that ships with Stateflow.

 

There are two Parameter name search mechanisms.  You can search and modify parameters using prompt strings.  This search mechanism is useful when you know the parameter by its dialog prompt string, but you don't know the name of the underlying attribute.  You can also perform a “fuzzy” search using property and/or value pairs.  This search mechanism is very useful for isolating the name of an underlying attribute in the absence of any other knowledge.

Running the Model Assistant Tool

At the MATLAB prompt, type

 

>> modelassistant('model') % invoke the tool for the root system of a model

>> modelassistant('system')  % invoke the tool for a particular system

 

You can conveniently invoke the Model Assistant Tool for models and systems using the built-in Simulink bdroot, gcb, and gcs commands.  For example

 

>> modelassistant(gcb)

 

Invokes the Model Assistant Tool for the currently select block (normally a subsystem).

Recommended Usage

The Model Assistant Tool is designed to complement Real-Time Workshop Embedded Coder, and works most effectively with ERT and ERT-based targets (such as MPC555), although it will operate with other targets.  Select the target you wish to generate before using the tool.

 


 


There are three demo model for the Model Assistant: advisordemo1, advisordemo2, and advisordemo3.  Type modelassistant('advisordemo1') to launch the advisordemo1 model with the tool, and follow these basic steps.

 

1.     Select the appropriate system target file.

2.     Invoke the Model Assistant Tool from MATLAB: modelassistant(model)

3.     Run the General Code Generation Goals component.

4.     Run the Model Advisor component, and address any reported issues.

5.     Adjust specific options using the Detailed Code Generation Goals component.

6.     Generate code for the model or subsystem of interest.
