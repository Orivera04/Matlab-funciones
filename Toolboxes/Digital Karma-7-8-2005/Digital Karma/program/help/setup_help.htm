Setting up the calculation:

1-D

This is basically a single row of cells to which you’re applying a rule set & saving each iteration as a large matrix.  You enter how many rows to save, how many iterations to run, the delay between iterations & the scale for color values.  Cellular automata are usually 0 to 1.  The initial value automatically sets the column size.

The initial value of the row can be set by typing in numbers separated by spaces or commas into the on-screen box.  You can also create more elaborate or longer initial rows by typing commands at the prompt such as

        initial=randint(1,20); %Generating 20 random 0’s & 1’s

        initial=zeros(1,300); initial(1,150)=1  %Generating 300 zeros with a one in the middle

Some initial values can be accessed by through File: Open Command Line Initial States.

Select or write a 1D rule set in the drop-box, then click Start

 

2-D

Mostly similar to above.  The rules now apply to the entire set of rows & each iteration adds to a multi-dimensional matrix.  2-D Initial States can be set two ways.  File: Open Command Line Initial States.  Use the 2-D initial state command lines at the bottom.

Or through excel.  Access though File: Open Excel Initial States.  This opens an excel file with saved custom initial states.  Clicking File: Load 2-D Initial State from Excel will load the FIRST worksheet in that excel file.  You can drag the worksheets to re-order them.

The rows box is useless under 2-D since the entire matrix is shown.
