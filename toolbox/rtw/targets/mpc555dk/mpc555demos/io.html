Real_Time I/O Demo Model

Demonstration of MPC555 I/O device driver blocks and CAN support. A subsystem of the mpc555rt_io model can be run on target hardware. The mpc555rt_io model can communicate via a CAN channel with the mpc555rt_iohost companion model, running in Simulink.

This model serves as a reference when using I/O device driver blocks provided with the Embedded Target for Motorola MPC555. This model illustrates how to use MPC555 I/O blocks for digital input/output, Pulse Width Modulation (PWM), waveform measurement, analog input, and Controller Area Network (CAN). PWM and digital I/O are demonstrated via Modular I/O System (MIOS) and Time Processor Unit (TPU3).

You can generate code for the subsystem Target_IO and run it on MPC555 hardware.

How to open this demo:

Click the link or at the command line type mpc555rt_io.

Note: You must have Vector-Informatik CAN hardware and drivers in order to use this demo to exchange CAN messages with the companion model mpc555rt_iohost.

If you have a CAN board and Vector-Informatik CAN drivers installed, you can exchange CAN messages with the companion model mpc555rt_iohost. The subsystem code communicates, via a CAN channel, with the mpc555rt_iohost model, running in Simulink.

I/O Host Demo

The MPC555 I/O Host Demo is run as a Simulink simulation which is non-real-time. During simulation, this model can be used to view signals from the MPC555 when running the mpc555rt_io model subsystem code on the Phytec board.

The I/O Host demo model is a companion model that provides signal viewing using Simulink Scope blocks. The subsystem code communicates, via a CAN channel, with the mpc555rt_iohost model, running in Simulink.

To open this model, click the link or at the command line type

  • mpc555rt_iohost

Generate code for the mpc555rt_io demo subsystem and download it to an MPC555. Configure the mpc555rt_iohost demo to look at a physical CAN channel by locating and double-clicking the Vector Configuration block and selecting a CAN channel under Channel parameter. Run the mpc555rt_iohost model in simulation and view the signals from the code running on the MPC555.
