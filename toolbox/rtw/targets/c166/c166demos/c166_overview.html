Embedded Target for Infineon C166® Microcontrollers Demonstrations

The Embedded Target for Infineon C166® Microcontrollers enables you to run Real-Time Workshop Embedded Coder generated code on a device belonging to this family of microcontrollers.

Core capabilities include:

  • Automatic build, download and run on target hardware
  • Automatic build and launch debugger
  • Single-tasking or multi-tasking execution without real-time operating system
  • Support for C166 bit-addressable memory
  • Report generation with RAM/ROM measurements

The Embedded Target for Infineon C166® Microcontrollers supports rapid prototyping and deployment of your application, as well as code validation and performance analysis.

See the demos for examples of automatic code generation and download to the target, real-time I/O models, the use of C166 bit-addressable memory, integrating hand-coded I/O functions, multitasking, execution profiling and the use of fixed point for efficient code generation.

Block Libraries

The Embedded Target for Infineon C166® Microcontrollers includes an I/O driver library that lets you add I/O driver blocks to your controller model for interfacing to hardware. The blocks are contained in the following sublibraries:

  • Asynchronous/Synchronous Serial Interface

    You can add Serial Transmit and Receive Blocks to your controller, to configure the C166 for serial communications.

  • CAN Interface

    You can add CAN Message Blocks to your controller, enabling you to send CAN messages containing signal data from your controller subsystem using either standard or extended CAN identifiers.

  • Execution Profiling

    Execution Profiling blocks provide an interface to the execution profiling engine over serial or CAN. On receipt of a start command message, logging of execution profile data is commenced. On completion of a logging run, the recorded data is automatically returned via serial or CAN.

  • TwinCAN Interface

    For processor variants such as the XC16x, there is support for the TwinCAN module.

To open the driver libraries, click the link or at the command line type c166drivers.
