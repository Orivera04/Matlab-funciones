Rules/Calculations

The rules or calculations performed each iteration are as I see boundless.  I’m experimenting with standard cellular automata rules.  But the computer will do whatever you type in the m-script to the active cells iterating.  It could be interacting with other programs, or getting existing data if you’d like.  The values can be any scale.  The graph scales the colors to defined by your High & Low values.

Make sure you don’t choose 1-D rules for a 2-D simulation and vice versa.

 

 

Editing Rules

Choose Edit File from the File menu, or by right clicking, to edit the rule selected in the drop down box.

The rule marked custom is for experimentation.  “A” is the variable assigned in the program to the matrix you’re graphing.

The rule is the entire instruction you want done to the matrix each time.  A simple example is adding one to a one dimensional progression.

a(end,:) = a(end,:) + 1;

The program will already have duplicated the last line in expectation of performing some alteration to it for the next.

Say you had [1,2,3]

You want to add one to teach number creating a new row.  Digital Karma will already have duplicated the last row so you have

[1,2,3; 1,2,3]

It will then run the above rule which says take the last row, a(end,)

And make it equal to the last row plus one, generating [1,2,3; 2,3,4]

Voila!

The program will chop off the top of the matrix according to how many rows you ask to save.  If you asked for only one it would now just be

[2,3,4]

New rules you want to be saved to the program by editing the code, but its simple.

Save the rule as an m-file in the directory.  Subdirectories don’t work.

They all start with z to keep together since not in a subdirectory.

To add them as defaults, edit chooser.m

Finally, type GUIDE in the matlab workspace to edit GUI named CA_GUI.fig

Double click on the rule drop box(1D or 2D), then STRING and add a name to the bottom which corresponds to what you put in chooser.m

Save your improvements to CA_GUI.fig & export it to digitalkarma.m.  You can also just double click CA_GUI.fig to run the program
