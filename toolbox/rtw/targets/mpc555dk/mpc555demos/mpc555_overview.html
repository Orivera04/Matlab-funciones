Embedded Target for Motorola® MPC555 Demonstrations

The Embedded Target for Motorola MPC555 enables you to run Real-Time Workshop Embedded Coder generated code on an MPC555 microcontroller.

Core capabilities include:

  • Real-time execution including MPC555 I/O drivers
  • Processor-in-the-Loop cosimulation
  • Report generation with RAM/ROM measurements
  • Automatic build, download and run on target hardware

The Embedded Target for Motorola MPC555 supports rapid prototyping and deployment of your application, as well as code validation and performance analysis.

See the demos for examples of automatic code generation and download to the target, real-time I/O and CAN messaging, processor-in-the-loop cosimulation, multitasking and execution profiling.

Block Libraries

The Embedded Target for Motorola MPC555 includes several driver libraries that let you add I/O driver blocks to your controller for interfacing to hardware. The top level library contains sublibraries and two blocks:

  1. The MPC555 Resource Configuration block. This supports driver configuration for MPC555 and MIOS, QADC, TPU and TouCAN submodules.
  2. The Watchdog timer block. In the event of application failure, this times out and resets the processor.
  3. All other driver blocks are contained in the following sublibraries: CAN, Execution Profiling, Interrupts, MIOS, QADC, Serial and TPU.
    • CAN 2.0B Controller Module (TouCAN) Sublibrary

      You can add CAN Message Blocks to your controller, enabling you to send CAN messages containing signal data from your controller subsystem using either standard or extended CAN identifiers.

    • Execution Profiling Sublibrary

      Execution Profiling blocks provide an interface to the execution profiling engine over serial or CAN. On receipt of a start command message, logging of execution profile data is commenced. On completion of a logging run, the recorded data is automatically returned via serial or CAN.

    • Interrupts Sublibrary

      For reading or writing signals attached to Asynchronous subsystems, driven by interrupt function call triggers.

    • Modular Input/Output System (MIOS1) Sublibrary

      You can add MIOS Blocks to your controller, enabling you to use digital I/O via the MIOS 16-bit parallel port I/O submodule, the MIOS Pulse Width Modulation Submodule, and support pulse width and pulse period measurement via the MIOS Double Action Submodule.

    • Queued Analog-to-Digital Converter (QADC) Module-64 Sublibrary

      You can add QADC Blocks to your controller. The QADC Digital In block enables you to use Queued Analog-Digital Converter (QADC64) pins as digital inputs, and the QADC Analog In block enables you to use the QADC64 in continuous scan mode.

    • Time Processor Unit (TPU3) Sublibrary

      You can add TPU driver blocks to your controller, enabling you to use the TPU modules for digital I/O, fast quadrature decode, pulse width modulation, new input capture/input transition counting, and the programmable time accumulator.

    • Serial Communications Interface (SCI) Sublibrary

      You can add Serial Transmit and Receive Blocks to your controller, to configure the MPC555 for serial communications on either of the QSMCM submodules SCI1 or SCI2.

    To open the driver libraries, click the link or at the command line type mpc555drivers.
