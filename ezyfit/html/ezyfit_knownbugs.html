EzyFit Known software problems


  • Compatibility Problems between EzyFit and MathWorks' Curve Fitting Toolbox.
    Both EzyFit and MathWorks' Curve Fitting Toolbox use a function named fit, so that the two toolboxes cannot be used simultaneously. If you want to have the two toolboxes installed and switch between the two, you just need to invert the order of the directories in the Set Path dialog box and to restart Matlab (Matlab searches functions sequentially, starting from the upper directory).

  • New user defined fits created from a given window are not automatically updated in the EzyFit menu of other window(s).
    Select the other window(s) and type efmenu to refresh the EzyFit menu.

  • When loading a fig file which already contains the EzyFit menu, a second EzyFit menu is added.
    Type efmenu, this will remove the extra menu.

  • Sometimes the menus disappear (except the first menu File) when clicking on the legend button.
    No idea for the moment. This bug seems to occur only with the R14 SP3 version (Matlab 7.1). Type efmenu to refresh the EzyFit menu.

  • Sometimes showslope displays a line with a slope slightly different from the required one.
    This problem originates from the time delay between the mouse click and the pointer location inquiry, especially when the mouse is moved quickly. Move slowly.

  • Sometimes selectfit displays the error message 'Error while evaluating figure WindowButtonUpFcn'.
    No idea for the moment. You may ignore these messages.

  • Selectfit does not keep the location of the legend box.
    This bug originates from drawpolygon, which returns wrong point location when a legend box is present. To avoid this problem, selectfit first closes the legend box and re-opens it at the end of the selection, so the legend box location is lost. No solution for this problem.

  • Selectfit does not work with docked figure.
    This bug originates from drawpolygon. Undock your figure.

  • You have found a bug not documented here?
