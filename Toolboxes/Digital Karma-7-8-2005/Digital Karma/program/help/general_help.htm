Window switch saves a, initial state, zoom, iteration, low, high

Also zoom back, undo, redo.  But not in switch window file. Switch window was integrated into their own files.

Not rule, input value or anything on menu bar

 

Last selection is primary in compare selections, first opened is primary as labeled in state comparison

 

 

Tools

Values and Click Changer:

Values Display: 

Displays in the output box at bottom of the screen: that cell’s value & coordinates, equivalent value of the row’s digits in the lowest base possible according to the values and the row’s values

Click Changer:  

Left or Right clicking activates the click changer when over the graph.  It will change that cell to the value in the “Changer” box at the left.

You can only change the active state.  The final row(1-D) or final matrix(2-D).  This keeps you from altering the iteration’s history.  This safety feature can be deactivated in Settings.

****There is currently a small bug that activates the click changer outside the graph when you are zoomed in.  So don’t right click outside the graph when zoomed in or you could change a value outside the zoom range without knowing.****

 

 

None:

Makes mouse clicking have no effect on the graph.  Normally deactivated, you can right-click over the graph as well in this mode

 

Compare 2 States-Detail:

Compare 2 states of equal size & iteration #.  First loaded state is original, 2nd is the updated state.  All different values are color coded.  If your compared graphs contain 0,1&2 then a changed value that is now 0 in the 2nd state but wasn’t before to a 3.  A changed value that is now 1 becomes 4 and a 2 becomes 5.  So 0,1,2 are the Normal values. Unchanged cells will still have those values.  Cells of 0,1&2 in the 2nd state that are different in the original are 3,4,5 respectively.  This way you see what cells changed & to what.  The encoding is dynamic.  If you have 10-19 in the original, changed cells after compare will be 20-29.  The high, low & scale also adjusts dynamically for the best setting w the new state generated by the compare.

****I didn’t make the scale turn on by default, but its helpful to use****

 

Compare Unlimited States-No Detail:

Compares unlimited number of states.  With 2 states there is an original & updated.  With more there is no final state.  A cell could be 0 in state one, 5 in state two, 10 in state 3, 6 in state four.  What is the “current” cell’s value.  So in this comparison all cells not identical throughout the states are made the value one higher than their cumulative maximum.  The settings all update dynamically as in a 2 state comparison.  You enter how many states to compare in the beginning so it know when to stop asking.

 

Delete Iterations:

Deletes iterations from the bottom.  Basically backing up the iteration.

Different from just scrolling back, deleting allow you to make a change a step X

and see how it effects the outcome.  You could save the state, delete some rows,

make a change, continue the iteration the number of deleted steps, then compare the two states

 

Crop: 

Crops any square from a 1-D sequence and any square and iteration number from a 2-D

 

Find and Replace:

For Find enter “a” then a logical statement.  a==1, a>0 a>3 | 2>a(everything but 3—2), a~=0(not 0)

The replace with should be a single value.

 

Undo: Crop, Delete Iterations Find and Replace & Click Changer all have undo.

 

Settings

Select 1-D or 2-D & grayscale or color

 

Force to Square:

Keeps the cells from getting stretched to cover the graph space.

 

Display values:

Shows the numerical values in the matlab workspace.  This works best for small arrays at slow speeds.  Alternately you can use the value Mouse over values under Tools.

 

Show Scale:

Displays a key for color to number.  Suppress Graph:  Speeds up the iteration by not displaying each step until turned-off.  You can then scroll backward to see the graph.

 

Limit Iterations Saved:

Saves only the number of iterations designated.  This is good for keeping up the speed of huge iterations or for conducting continuous iterations where you pick up next time from one of the states iterated to.  2-D iterations especially become slow with large iterations.

 

History Unchangeable/Allow History Change:

Engages & deactivates Click Chagner safety feature regarding altering iteration’s past.  More details under Click Changer

 

File & Action Menu:

Most functions here are self-explanatory.  Below are some notes

1-D Initial States:

List of command line functions for setting up initial states.  They’re not buil-in to the menu bar so you can easily alter them after pasting into Matlab workspace.

 

2-D Initial States:

The first worksheet in the workbook will be imported as the 2-D initial state.  The initial state can ONLY be edited here.  Command line operations will be over-written.

 

Edit Rule:

Shows the actual file with the selected rule.  Changing this file will alter the actual rule.  To experiment, copy the rule, select Custom from the rule drop-box, and paste it there.

 

Record Avi:

Has a bug & doesn’t work currently.

 

Other

“a” is the variable holding the matrix of values.(Matlab is case sensitive)

1-D A uses a one dimensional matrix.  So a(1:5, 3:9) gives you the 1st—5th rows & 3rd—9th columns of the matrix.  2-D uses a muti-dimensional array.  a(:,:,1:4) gives you the matricies for the initial state through the 3rd iteration.

 

Most features can be changed on the fly.  The dimension(1-D/2-D) should be changed when starting a NEW iteration and Iterations changed during a Pause or Start New.  Otherwise you can switch any Settings right in the middle of iterating.

 

Display:

Shows the current scroll value.  Typing in it can move more precisely than the scroll bar.  When “Limit number of iterations saved” is off, the display value equals the number of iterations saved.
