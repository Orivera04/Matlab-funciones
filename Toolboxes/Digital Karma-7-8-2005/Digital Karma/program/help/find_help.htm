Find Help

Quote marks enclose variables in this document and should be omitted when used.

 

Basics

‘findarea’ stands for the area being searched.

Set under ‘Analysis Area’ either the entire matrix, what is currently graphed or a selection.

Find can search for scalars and matrices.

All matrices must be the same size, scalars are expanded to matrix size.

 

Example search statements

findarea==4                           any cell equal to four

findarea==[0,1;1,2]

findarea==cat(3,[1,1],[2,2])  cat creates multi-dimensional arrays, which can be used for searching

findarea==findarea               the entire search area

findarea>=7 | findarea<5      any value greater than or equal to 7 or  less than 5.

findarea<1 & findarea~=0    less than 1 and not 0

findarea>=[1,2;3 4] & findarea<7    The seven will be expanded

Example OR statement as explained below

zero_round(findarea==[0 1 0])   |   zero_round(findarea==[1 0 1])

 

 

Wildcard Searches

Say you want to search for any occurrence of:1

                                                              2 0 2

But you don’t care what values are next to the 1.  You’re searching for the triangle pattern.

As far as I know, there is no wildcard place holder you could use like [ *,1,*; 2,0,2].

So you just search for the last row AND the 2nd value of the first row.

The simplest way is to copy one example of the matrix so you can use ‘matrixpaste’(as mentioned below) for addressing.

 

Enter in search box:  findarea(2,:)==matrixpaste(2,:) & findarea(1,2)==matrixpaste(1,2)

Type ‘matrixpaste’ in the lower box set the search size.

This will find every instance of the above triangle regardless of what is next to the one.

Effectively placing wildcards next to the 1.

           

 

The Input Box’s value is used for Replace

Scalars and matrices can be used.

‘aselection’ stands for what is selected.

You could use the current found selection to base the replace from with aselection*2

That would double each item found when you press replace.

 

‘matrixpaste’ is a variable which holds the last selection copied.

It can be typed into the input box to replace a selection with a previously copied selection.

 

‘a’ stands for entire matrix.

Entering a(1,2) in the input box will replace with that coordinate’s value.

For 1-D, you must add 1 to the row that is displayed since the 0th iteration is the 1st index row.

The same for the plane in 2-D, a(10, 20, 4) is cell [10, 20] in the 3rd iteration.

You would use a(10:11, 20:22) for a 2X3 matrix replace.

Scalars will NOT be expanded by Replace from within the Input Box.

 

 

Replace All

‘aselection’ won’t work since you are replacing everything at once.

A special variable for this case, ‘foundvalues’ was created.

foundvalues*+1 would add 1 to all values found during Replace All.

 

Special attention to the OR statement

Logical operators are applied to the result for each element's comparison.

That means findarea==[0 1 0]  |  findarea==[1 0 1] is interpreted as

findarea(1) can equal 0 or 1, findarea(2) can equal 1 or 0 and findarea(3) can equal 0 or 1.

When you wanted to say findarea can equal [0 1 0] or [1 0 1]

But the or is applied to each element in the matrices individually by Matlab.

 

Encase each statement with the “all” function to output a single 0 if any element is false.

all(findarea==[0 1 0])   |  all(findarea==[1 0 1])

Now only arrays that match one of the two completely are found.

 

Find Next within a selection

Find Next with the Analysisarea set to ‘selected values’ works by making the selection, then doing the find.

Then reselect the original selection & push Find Next.  It will then get the next occurrence in the selection.

So you just have to reselect each time, but you can limit it to a selection then and still use Find Next.
