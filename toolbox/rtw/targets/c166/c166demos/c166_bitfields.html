C166 Bit-Addressable Memory Demo

The C166 has special instructions for accessing bit-addressable memory. For variables that are declared as bit-addressable, the compiler is able to generate assembler code using these special instructions, thereby resulting in reduced code size and faster execution. This example shows how to configure your model to take advantage of bit-addressable memory.

  1. To open this model, click the link or at the command line type c166_bitfields
  2. Double click the block Generate code and launch debugger.
  3. Double click the block Read documentation for C166 bit-addressable memory to see a tutorial explaining this example in detail in the User's Guide, "Custom Storage Class for C166 Microcontroller Bit-Addressable Memory".
